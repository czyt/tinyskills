## 🆕 新增功能

### 1. 智能 compose_override 生成

**偏好 ID**: `generate_compose_override`  
**默认值**: `true`

#### 功能说明
- 自动检测是否需要 `compose_override` 配置
- 仅在有需要覆盖的 Docker Compose 参数时才生成
- 支持设备挂载、GPU 加速、特殊网络配置等场景

#### 智能检测规则
```python
需要 compose_override 的场景：
1. 挂载主机设备 (devices)
2. 挂载非 /lzcapp 路径的 volumes
3. GPU/USB/KVM 加速配置
4. 特殊网络配置
```

#### 示例输出

**有需要覆盖的参数时**:
```yaml
# lzc-build.yml
manifest: ./lzc-manifest.yml
pkgout: ./
icon: ./icon.png

compose_override:
  services:
    app:
      devices:
        - /dev/ttyUSB0:/dev/ttyUSB0
      deploy:
        resources:
          reservations:
            devices:
              - driver: nvidia
                count: 1
                capabilities: [gpu]
```

**无需覆盖时**:
```yaml
# lzc-build.yml
manifest: ./lzc-manifest.yml
pkgout: ./
icon: ./icon.png

# 无 compose_override 部分
```

### 2. 后台任务模式 (background_task)

**偏好 ID**: `background_task`  
**默认值**: `false`

#### 功能说明
- 为常驻后台服务自动设置 `application.background_task: true`
- 防止系统因长时间无活动而自动休眠应用
- 支持智能检测（基于镜像名称关键字）

#### 适用场景
- 下载器 (aria2, qBittorrent)
- 同步服务 (Yuque Sync, RSS sync)
- 定时任务 (cron jobs)
- 需要 24/7 运行的后台服务

#### 智能检测关键字
```python
background_keywords = [
    'sync', 'scheduler', 'cron', 'worker',
    'downloader', 'aria2', 'qbittorrent',
    'background', 'daemon'
]
```

#### 示例输出
```yaml
application:
  subdomain: yuque-sync
  background_task: true  # 防止自动休眠
```

### 3. 默认 public_path 配置

**偏好 ID**: `add_root_to_public_path`  
**默认值**: `true` (已存在，本次更新完善)

#### 功能说明
- 自动为应用添加 `/` 到 `public_path` 数组
- 适用于提供静态文件或需要根路径访问的应用

#### 示例输出

**启用时** (默认):
```yaml
application:
  subdomain: myapp
  public_path:
    - /
  upstreams:
    - location: /
      backend: http://myapp:8080/
```

**禁用时**:
```yaml
application:
  subdomain: myapp
  upstreams:
    - location: /
      backend: http://myapp:8080/
```

### 4. 包名前缀选择与偏好记忆

**偏好 ID**: `package_prefix`  
**默认值**: `cloud.lazycat.app`

#### 功能说明
- 允许用户选择应用包名前缀
- 记忆用户选择，后续自动使用
- 支持三种模式 + 自定义

#### 可用选项
1. `cloud.lazycat.app` - 官方或认证应用 (默认)
2. `community.lazycat.app` - 社区贡献应用
3. 自定义前缀 - 企业或个人命名空间 (如 `com.acme.app`)

#### 工作流程
```
首次使用 → 提示选择前缀 → 保存偏好
后续使用 → 自动应用保存的前缀
用户可随时更改 → 更新偏好
```

#### 示例输出
```yaml
# cloud.lazycat.app (默认)
package: cloud.lazycat.app.myapp

# community.lazycat.app
package: community.lazycat.app.myapp

# 自定义前缀
package: com.acme.app.myapp
```

## 🎯 使用指南

### 配置偏好

在 Claude Code 的 skill 偏好设置中配置：

```
Settings → Skills → lazycat-app-publisher → Preferences
```

### 偏好设置建议

| 偏好 | 推荐值 | 适用场景 |
|------|--------|----------|
| `package_prefix` | `cloud.lazycat.app` | 官方应用 |
| `package_prefix` | `community.lazycat.app` | 社区应用 |
| `package_prefix` | 自定义 | 企业应用 |
| `background_task` | `true` | 后台服务 |
| `background_task` | `false` | Web 应用 |
| `generate_compose_override` | `true` | 需要特殊配置 |
| `add_root_to_public_path` | `true` | 提供静态文件 |

## 💡 实际应用示例

### 示例 1: 后台同步服务

```yaml
# lzc-manifest.yml
name: Yuque Sync
package: community.lazycat.app.yuque-sync  # 使用 community 前缀
version: 1.0.0

application:
  subdomain: yuque-sync
  background_task: true  # 自动检测或用户设置
```

### 示例 2: GPU 加速应用

```yaml
# lzc-build.yml
manifest: ./lzc-manifest.yml
pkgout: ./
icon: ./icon.png

compose_override:  # 自动检测到 GPU 需求
  services:
    ai-app:
      deploy:
        resources:
          reservations:
            devices:
              - driver: nvidia
                count: 1
                capabilities: [gpu]
```

### 示例 3: 普通 Web 应用

```yaml
# lzc-manifest.yml
name: My Web App
package: cloud.lazycat.app.mywebapp
version: 1.0.0

application:
  subdomain: mywebapp
  public_path:  # 自动添加
    - /
  upstreams:
    - location: /
      backend: http://web:8080/

# lzc-build.yml
manifest: ./lzc-manifest.yml
pkgout: ./
icon: ./icon.png
# 无 compose_override (不需要)
```
